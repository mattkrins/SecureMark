SecureMark Privacy & Security

Our Promise:

  1. SecureMark does not store any logs, period. We will never track or trace anything, ever.
  2. This extension is entirely client-side and only communicates with the chrome bookmarks API.
  3. We cannot be compelled by law to provide identifying information because we have none.
  4. Encrypted bookmarks are stored as HEX data using the CryptoJS library with the AES cypher suite - the AES algorithm is still considered completely secure & unbroken.
  5. The SecureMark GitHub repository will always be identical to the published version.

INPORTANT - PLEASE READ:

  • Although SecureMark does not store any logs, there is no way of knowing what Google does.
  • Encrypted data is impossible to recover without the password. If you forget it, goodbye data.
  • Always backup your bookmarks before using this extension. In the unlikely event that a bug occurs and your data is encrypted incorrectly, it is unlikely to be recoverable.
  • We provide no support, warranty or hold any liability for issues arising from use of this extension. Use at your own risk.
  • If you find a bug or issue with SecureMark, you can report it via the chrome store support link.

Uh-Oh!

Forgotten your password?

SecureMark does not store passwords and so encrypted bookmarks are completely unrecoverable.

You will need to open the chrome bookmark manager and delete your encrypted bookmarks.
